# 記事執筆ガイド

## Zenn

### 読者像
- エンジニア・開発者
- 技術的な再現性・正確さを求めている
- コードや構造を重視する

### 求められること
- **再現性**：読者が同じ手順で試せること
- **正確さ**：技術用語・コードに誤りがないこと
- **構造の明確さ**：何を学べるか最初にわかること
- **コードブロック**：コマンドやコードは必ずコードブロックで記載

### 記事の型
```
## この記事でわかること（or 概要）
## 環境・前提条件
## 手順（番号付き）
## つまづきポイントと解決法
## まとめ
```

### frontmatter（必須）
```yaml
---
title: "タイトル"
emoji: "🎵"          # 記事のアイコン絵文字
type: "tech"         # tech（技術）or idea（アイデア）
topics: ["react", "python"]  # 最大5つ、小文字英数字
published: false     # 公開時にtrue
---
```

### 使えるZenn独自記法
```
:::message
補足・メモ（青）
:::

:::message alert
注意事項（赤）
:::

:::details 折りたたみタイトル
折りたたみ内容
:::
```

### URLの貼り方
URLを単独行に貼るとカード表示になる（Zenn推奨）
```
https://example.com
```

### NG
- 主観的な感情表現が多すぎる
- 手順が曖昧で再現できない
- コードがコードブロック外にある

## Note

### 読者像
- 医療者・看護師
- プログラミング未経験〜初心者
- 「作りたいけどわからない」人
- ゆーのさんの成長を応援している人

### 求められること
- **共感**：「わかる！」と思ってもらえる失敗談や感情
- **わかりやすさ**：専門用語を自分の言葉で噛み砕く
- **ストーリー性**：なぜやったか・何が起きたかの流れ
- **親しみやすさ**：医療者としての視点・例えを活かす
- **画像**：スクショや図があると伝わりやすい

### 記事の型
```
## 背景・動機（なぜやろうとしたか）
## やってみた（何が起きたか）
## 学んだこと
## 次回予告
```

### 書くときのポイント
- 失敗・つまづきを隠さず書く（それが一番刺さる）
- 医療の例えを使う（わかりにくい技術をナース視点で翻訳）
- 「なんでこうなるの？」という素朴な疑問を大切にする
- 感情の流れ（困惑→試行錯誤→解決→次へ）を意識する

### タグの使い方
読者に届けたい属性を意識して選ぶ
```
#プログラミング初心者 #個人開発 #看護師 #アプリ開発 #学習記録
```

### NG
- 技術的な正確さにこだわりすぎて読みにくくなる
- 感情がなく淡々としすぎる
- 読者置いてけぼりの専門用語多用

## ZennとNoteの使い分け

| | Zenn | Note |
|---|---|---|
| 主な内容 | 技術手順・学習記録（技術側面） | 開発ストーリー・感情・背景 |
| 文体 | 整理・構造重視 | 会話・ストーリー重視 |
| コード | 詳しく載せる | 雰囲気だけ、詳細はZennへ |
| 画像 | スクショ（手順説明用） | スクショ＋感情を表す画像 |
| 同じ記事を両方に | 技術部分をZenn、体験部分をNoteに分けてもOK | ← |

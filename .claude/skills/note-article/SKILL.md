# Note 記事作成ガイド

## 読者像
- 医療×テックに関心がある人
- プログラミング未経験〜初心者
- 音楽療法関係者
- 個人開発者
- ゆーのさんの成長を応援している人

## Noteで求められること
- **共感**：「わかる！」と思ってもらえる失敗談や感情
- **ストーリー性**：なぜやったか・何が起きたかの流れ
- **わかりやすさ**：専門用語を自分の言葉で噛み砕く
- **親しみやすさ**：医療者としての視点・例えを活かす

## 記事執筆前（企画段階）

### テーマ・目的

- [ ] この記事で伝えたい**メインメッセージ**は明確か？
- [ ] **読者ペルソナ**（誰に向けて書くか）を決めたか？
- [ ] 読者がこの記事から得られる**価値**は何か？

### キーワード設定

- [ ] 検索されやすいキーワードを3-5個リストアップしたか？

## 執筆中

### タイトル設計の4つの問い

> **誰に × どんな価値 × どんな内容 × どう伝えるか**

| 問い | 具体例 |
|------|--------|
| ① 誰に向けた内容か？ | 副業初心者・ASD特性持ち・看護師 |
| ② どんな価値があるか？ | 3ヶ月で初収益・変われない理由がわかる |
| ③ どんな内容か？ | 音楽療法の基本・3ステップの実践 |
| ④ どう伝えるか？ | 体験談・ステップ解説・ビフォーアフター |

**避けたいタイトルパターン：**

| ❌ 避けたい例 | ⭕ 改善後 |
|-------------|---------|
| 最近思ったこと | 〇〇を始めて3ヶ月で気づいた、△△の考え方 |
| 開発日記 #1 | 看護師がReactでアプリ開発を始めた理由と最初の壁 |

### 構成の型

```
背景・動機（なぜやろうとしたか）
  ↓
やってみた（何が起きたか・失敗も含めて）
  ↓
学んだこと
  ↓
次回予告
```

### 本文チェック

- [ ] スマホで読みやすい改行か（2-3文ごとに改行）
- [ ] 失敗・つまづきを隠さず書いているか（それが一番刺さる）
- [ ] 専門用語は簡単な言葉で説明したか

## ファイル管理ルール

- **ZennのfrontmatterはNoteファイルに不要**。`title` / `emoji` / `type` / `topics` / `published` のfrontmatterブロックは削除する
- Zenn版とnote版は**タイトルが異なる場合がある**（Zenn版は技術寄り・note版は読者に刺さるタイトル）
- noteには独自のfrontmatter（`platform: note` / `published_date` / `tags`）を使うか、frontmatterなしでよい

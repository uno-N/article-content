# Zenn 記事作成ガイド

## 読者像
- エンジニア・開発者
- 技術的な再現性・正確さを求めている
- コードや構造を重視する

## Zennで求められること
- **再現性**：読者が同じ手順で試せること
- **正確さ**：技術用語・コードに誤りがないこと
- **構造の明確さ**：何を学べるか最初にわかること
- **コードの丁寧さ**：コマンドやコードは必ずコードブロックで

---

## 記事執筆前

- [ ] この記事で伝えたい**技術的な学び**は明確か？
- [ ] `topics`（タグ）を1〜5個決めたか？（小文字英数字・例：react, python, api）

---

## 執筆中

### frontmatter（必須）

```yaml
---
title: "タイトル"
emoji: "🎵"
type: "tech"            # tech or idea
topics: ["react"]       # 最大5つ、小文字英数字
published: false
---
```

### タイトル

- [ ] **何ができるようになるか**が伝わるタイトルか？
    - ❌ 「環境構築した」
    - ⭕ 「React環境構築の全記録 - つまづきポイントと解決法まとめ」

### 構成の型

```
## この記事でわかること
## 環境・前提条件
## 手順（番号付き）
## つまづきポイントと解決法
## まとめ
```

### 本文チェック

- [ ] コマンドはすべてコードブロックに入れているか？
- [ ] コードブロックに言語名を指定しているか？（```bash, ```python, ```jsx など）
- [ ] 手順が飛んでいないか確認したか？

### Zenn独自記法

```
:::message
補足・メモ（青色ボックス）
:::

:::message alert
注意事項・警告（赤色ボックス）
:::

:::details 折りたたみタイトル
折りたたみ内容
:::
```

## NoteとZennの使い分け

- **体験・感情・ストーリー** → Note
- **手順・コード・技術解説** → Zenn
- 同じ開発体験でも両方に書けるネタがある

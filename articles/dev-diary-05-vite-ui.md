### 0-2. そもそもCRAやViteってなんだっけ？
前提知識：環境構築について詳しくはこちらをお読みください。

Viteとは、ビルドツールのこと。

前回の図でいうここの位置にいます。
```
Node.js（JavaScriptランタイム）
 ├── V8エンジン
 ├── libuv
 ├── Webpack（Reactコード変換）  ← ここをViteが置き換える
npm（Node Package Manager）
      └── react

```
じゃあ、ビルドツールって何だっけ？

ビルドツール = 開発者が書きやすいコード(Javascriptなど)を、ブラウザが動かせる形に変換・最適化するもの

詳しい機能：

- コードを圧縮、軽量化して読み込みを速くする
- ブラウザが読める形式に変換する
- コンポーネントで書いたJSXから複数ファイルをまとめる(バンドル)

[![Image from Gyazo](https://i.gyazo.com/9580393d3fee1cf716a17da11512624b.png)](https://gyazo.com/9580393d3fee1cf716a17da11512624b)


以下コマンドで実施。
```bash
# 旧：Create React App（非推奨）
npx create-react-app my-app

# 新：Vite（こっちを使おう）
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev
```

特に大きな問題もなく乗り換えられました。

## 1. 診断フェーズの実装に取り掛かる

このアプリを作るにあたって書いたMVP仕様書があります。

全然違うものが出てきた。

**そりゃそうだ。パス指定してなかった。**

Claude Codeはファイルを自動で読みに行くわけではなく、「このファイルを参照して」と明示的に伝えないといけません。

パスとは？
ファイルは**階層構造（フォルダの中にフォルダ）**で管理されています。

```
Desktop/
 └── neiro-project/
      ├── src/
      │    └── Aファイル  ← このファイルに辿り着きたい
      └── mvp-spec.md
```
このようなフォルダ構造の際、このとき Aファイル のパスは：

```
/Users/yuno/Desktop/neiro-project/src/Aファイル
```

私が大学院で学んだ一番最初のプログラミング(?)の基礎です。urlも同じ仕様です。
パスを伝えることでAファイルに辿り着く時に、どの通り道でいくかがわかります。

パスのコピーは右クリックでこんな感じでできます。
[![Image from Gyazo](https://i.gyazo.com/61a9486ce6fb0c5f9e240810e82fce44.gif)](https://gyazo.com/61a9486ce6fb0c5f9e240810e82fce44)

パスを指定して再度依頼すると、ちゃんとMVP仕様書の内容を読み込んで、意図通りのコンポーネント構成を出してくれた。

## バズった要因の分析

**① 「非エンジニアです。」の一行目が最強**

冒頭の一言で「自分ごと化」のハードルを一気に下げた。DESIGN.mdの元ツイートは技術者向けに見えるのに、「非エンジニアでもできた」という実績証明になっている。

**② 元ツイートのバズに乗った**

@L_go_mrk（AI駆動塾）のDESIGN.md投稿自体が人気ツイート。そこへの引用なので、元ツイートのタイムラインに露出しながら、独自の「使用報告」という付加価値をのせられた。

**③ ブックマークがいいねを上回っている**

1,605 bookmarks > 1,145 likesというのは重要で、「後で自分も試したい」という実用的な保存動機が強く働いている。拡散じゃなく保存される＝情報の実用価値が高い証拠。

**④ 「そのまま渡す」というハック感**

「ClaudeCodeにそのまま渡す」という具体的な使い方がシンプルで再現性が高い。ハウツーとして完結している。

**⑤ 画像3の引用RTのコメントが象徴的**

「もうアプリやソフト作れることでエンジニアを名乗れる時代は終わった」というコメントが来ている＝ポスト自体が「AI時代の非エンジニア像」という大きな議論を呼ぶ文脈に乗った。

## 5. まとめ

### 5-1. 今回の学び

- ✅ **CRAは非推奨 → Viteに乗り換えた**：ビルドツールが変わるだけで、開発サーバーの起動が体感できるほど速くなった
- ✅ **Claude Codeにはパスを明示的に渡す**：「仕様書を読んで」だけでは動かない。`/Users/yuno/.../mvp-spec.md` と住所まで指定して初めて読んでくれる
- ✅ **「いい感じに」はDESIGN.mdで具体化できる**：有名プロダクトのデザイン仕様書を渡すと、一貫したスタイルで実装してくれる
- ✅ **デザインは試して比べるのが一番早い**：3スタイル試して、合体させたら納得のUIができた

### 5-2. つまずきポイントと解決法

**MVP仕様書が反映されない：**
Claude Codeにファイルを読んでほしい時は、ファイルパスを明示して渡す。「なんとなく実装して」ではなく「この仕様書を読んで実装して」。

### 5-3. 次回の挑戦は、、？

音楽生成フェーズへの接続。
診断フェーズで収集した気分・エネルギー・テンポ・性格タイプを、Lyria RealTime APIのパラメータに変換して、実際に音楽を生成する。

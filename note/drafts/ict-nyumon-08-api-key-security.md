## この記事でできるようになること

- APIキーをコードに直接書いてはいけない理由がわかる
- `.env` ファイルに秘密情報を書く方法がわかる

**前提：Git・GitHubの記事を読んでいればOK。**

---

## なぜAPIキーをコードに書いてはいけないのか

こういうコードを書いたとします。

```javascript
const apiKey = "sk-abc123xyz456...";   // ← APIキーを直接書いている
```

このファイルをGitHubにアップすると——パブリックリポジトリなら全世界に公開されます。

APIキーは「サービスを使う権限のパスワード」です。流出すると：

- 他人に使われて料金が発生する（1日で数万円の被害も）
- サービスを悪用される
- キーを無効化して再発行が必要になる

患者IDやパスワードをカルテに生で書いて廊下に貼り出すようなものです。絶対にやってはいけない。

## 3ステップで守る

### ステップ①　`.env` ファイルにAPIキーを書く

プロジェクトのフォルダに `.env` という名前のファイルを作ります。

```bash
GOOGLE_API_KEY=あなたのキーをここに書く
OPENAI_API_KEY=あなたのキーをここに書く
```

`.env` は「環境変数」のファイルです。コードの中に直接書く代わりに、ここに書いておきます。

ファイル名の先頭に `.`（ドット）がついているのがポイントです。Macでは隠しファイルとして扱われます。

### ステップ②　`.gitignore` に `.env` を追加する

プロジェクトのフォルダに `.gitignore` というファイルを作ります（すでにある場合は開きます）。

```
.env
```

これだけ書いて保存します。

`.gitignore` は「Gitがアップロードしないファイルの一覧」です。ここに書いたファイルは `git add` の対象から外れます。

`git` ＋ `ignore（無視する）` ——読んで字のごとくです。

### ステップ③　コードから `.env` を読み込む

コードの中では、キーの値を直接書く代わりに「環境変数の名前」で参照します。

**JavaScript（Node.js）の場合：**

```javascript
const apiKey = process.env.GOOGLE_API_KEY;
```

`process.env.変数名` で `.env` に書いた値を取り出せます。

---

## `.gitignore` に書き忘れていないか確認する

すでにGitを使っているプロジェクトで確認します。

```bash
cat .gitignore | grep .env
```

`.env` が出てきれば設定済みです。何も出なかったら追加してください。

---

## 過去にコミットされていないか確認する

`.gitignore` に追加する前に誤ってコミットしていた場合、履歴に残っています。

```bash
git log --all -- .env
```

何も出なければOKです。何か出たら：

1. 今すぐAPIキーをローテーション（再発行）する
2. GitHub上のリポジトリの該当コミットから削除する

キーが漏洩した場合は「直す」より「無効化して新しいキーにする」が先です。

---

## まとめ

| やること | 方法 |
|---|---|
| APIキーを隠す | `.env` ファイルに書く |
| GitHubにアップしない | `.gitignore` に `.env` を追加 |
| コードから読み込む | `process.env.変数名` で参照 |
| 漏洩確認 | `git log --all -- .env` |

コードを書く前に一度だけやっておけば、あとは気にしなくていいことばかりです。

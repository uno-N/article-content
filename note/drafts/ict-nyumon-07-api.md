## APIとは何か

API（Application Programming Interface）は、プログラム同士が通信するための「窓口」です。

一番わかりやすいのが飲食店の例えです。

```
お客さん（アプリ）
    ↓ 注文（リクエスト）
ウェイター（API）
    ↓ 厨房に伝える
厨房（サーバー・外部サービス）
    ↓ 料理ができた
ウェイター（API）
    ↓ 料理を運ぶ（レスポンス）
お客さん（アプリ）が受け取る
```

アプリは厨房の中を知らなくていい。「これをください」と頼んで、結果を受け取るだけ。

医療で言えば、検査オーダーと同じ構造です。臨床側は「この検査をしてください」と依頼して、結果だけ受け取ります。検査機器の内部処理は知らなくていい。

---

## 具体例で見る

天気予報アプリを作りたいとします。

天気のデータを自分で集めるのは大変です。でも「天気APIサービス」を使えば、「東京の今日の天気をください」と送るだけで、データが返ってきます。

地図アプリ・翻訳・決済・AI——これらはすべてAPIとして提供されています。自分で作らなくていい。

---

## リクエストとレスポンス

APIとの通信は「リクエスト（要求）」と「レスポンス（応答）」のセットです。

**リクエスト**：「何をしてほしいか」を送る
**レスポンス**：結果が返ってくる

```
アプリ → [リクエスト] → APIサーバー
アプリ ← [レスポンス] ← APIサーバー
```

レスポンスは多くの場合、**JSON形式**で返ってきます。

```json
{
    "city": "Tokyo",
    "temperature": 22,
    "weather": "Cloudy"
}
```

`{ }` で囲まれた「キー：値」のペアです。`"city"` というキーに対して `"Tokyo"` という値が入っています。

---

## APIキーとは

APIサービスの多くは「誰がリクエストしているか」を確認するために**APIキー**を使います。

APIキーは「利用証」のようなものです。登録して発行してもらい、リクエストの中に含めて送ります。

```
アプリ → [リクエスト + APIキー] → APIサーバー
              ↑
         「このキーを持つ人に使わせてOK」と確認される
```

APIキーは**流出させてはいけません**。他人に使われると、自分のアカウントに課金されたりサービスを悪用されたりします。次の記事で安全な管理方法を書きます。

## 実際のAPIの呼び出し（コードで見る）

JavaScriptでAPIを呼び出す基本の書き方です。読めるようになれば十分です。

```javascript
fetch("https://api.example.com/weather?city=Tokyo")
    .then(response => response.json())
    .then(data => {
        console.log(data.temperature);  // 温度を表示
    });
```

- `fetch(URL)` — URLに向けてリクエストを送る
- `.then(response => response.json())` — 返ってきたデータをJSON形式に変換する
- `.then(data => { ... })` — データが届いたら何をするか

「送って、受け取って、使う」という3ステップです。

## まとめ

| 用語 | 意味 |
|---|---|
| API | プログラム同士が通信する窓口 |
| リクエスト | アプリがAPIに送る要求 |
| レスポンス | APIからアプリに返ってくる結果 |
| JSON | APIのレスポンスによく使われるデータ形式 |
| APIキー | 「このサービスを使う権限がある」ことを示す鍵 |
